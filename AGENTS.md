- **产品类型**: 办公工具类Web应用
- **场景类型**: <scene_type>prototype-app</scene_type>
- **目标用户**: 办公人士、企业员工、需要处理PDF文件的个人和商业用户

## 页面布局建议

> **说明**：5个功能页面采用统一的布局结构，仅操作参数区和处理逻辑不同

- **布局模式**: 左右分栏（Sidebar + 主内容区），主内容区上下分区（上传区 + 文件列表区 + 操作参数区 + 下载结果区）
- **视觉重心**: 上传区和文件列表区 —— 用户首先关注文件上传和管理，其次是操作参数配置
- **结果承载区**: 下载结果区（处理完成后显示下载按钮/列表）；初始态为上传引导空状态
- **源材料承载区**: 文件列表区（上传后展示文件缩略/名称/大小/页数，支持拖拽排序/删除）
- **顶栏**: 右上角显示激活状态标签 + 激活按钮 + 主题切换按钮
- **全局弹窗**: 首次打开显示激活弹窗，可关闭进入试用模式

## 功能列表

### 全局功能

- **页面目标**: 提供统一的应用框架、激活系统、主题切换和全局状态管理
- **功能点**:
  - **激活弹窗**: 首次打开应用时弹出激活窗口，包含激活码输入框、激活按钮、试用版进入按钮；已激活用户不再弹出
  - **激活状态管理**: 读取localStorage中的激活状态，顶栏显示"已激活"或"试用版"标签；试用版限制每次处理不超过3个文件，导出文件带试用水印
  - **激活码校验**: 校验格式为 `PDF-XXXXXXXX`（8位字母数字），最后一位为校验位（简单校验算法，如前7位ASCII码之和模10等于最后一位数字）
  - **主题切换**: 支持深色/浅色主题切换，切换动画流畅，偏好保存在localStorage
  - **玻璃拟态全局样式**: 统一的 Glassmorphism 风格（毛玻璃背景、模糊效果、半透明边框、柔和阴影）
  - **响应式布局**: 桌面端侧边栏展开，移动端侧边栏收起为汉堡菜单，主内容区自适应

### PDF合并页面

- **页面目标**: 用户上传多个PDF，调整顺序后合并为一个PDF下载
- **功能点**:
  - **拖拽上传区**: 支持点击选择文件和拖拽文件上传，支持多文件，仅接受PDF格式，上传时显示文件校验
  - **文件列表管理**: 显示已上传文件列表（文件名、大小、页数），支持拖拽调整顺序、删除单个文件、清空全部
  - **合并操作**: 点击合并按钮后显示进度条动画，使用 pdf.js 读取各文件页面，jsPDF 按顺序合并生成新PDF
  - **试用版限制**: 试用版最多上传3个文件，超出时提示；导出的合并PDF每页添加"试用版"水印
  - **下载结果**: 处理完成后显示下载按钮，点击下载合并后的PDF文件，文件名可自定义
  - **错误处理**: 文件损坏、读取失败等异常情况给出友好提示

### PDF拆分页面

- **页面目标**: 用户上传一个PDF，按页码范围或每页拆分，导出多个PDF
- **功能点**:
  - **拖拽上传区**: 支持单PDF文件上传，显示文件总页数
  - **拆分模式选择**: 两种模式切换——按页码范围拆分（输入起始-结束页，可添加多组范围）、按每页拆分（每页生成一个PDF）
  - **范围配置**: 范围模式下可添加/删除多个页码范围，支持输入校验（页码不超出总页数、起始≤结束）
  - **拆分操作**: 点击拆分按钮显示进度条，使用 pdf.js 读取指定页面，jsPDF 生成多个PDF文件
  - **试用版限制**: 试用版最多拆分出3个文件，超出时提示；每个导出PDF带水印
  - **批量下载**: 拆分完成后显示文件列表，支持单个下载或打包为ZIP批量下载

### PDF旋转页面

- **页面目标**: 用户上传PDF，选择页面和旋转角度，保存后下载
- **功能点**:
  - **拖拽上传区**: 支持单PDF文件上传，显示文件总页数
  - **页面选择**: 支持选择全部页面、指定页码范围、奇数页、偶数页等多种选择方式
  - **旋转角度选择**: 90度顺时针、180度、270度逆时针三个角度按钮选择
  - **预览缩略图**: 显示页面缩略图，可直观看到旋转效果（可选，首屏至少支持页码输入）
  - **旋转操作**: 点击应用按钮显示进度条，使用 pdf.js 渲染旋转后的页面，jsPDF 生成新PDF
  - **试用版限制**: 试用版最多处理3页，超出时提示；导出PDF带水印
  - **下载结果**: 处理完成后显示下载按钮

### PDF加密页面

- **页面目标**: 用户上传PDF，设置打开密码，加密后下载
- **功能点**:
  - **拖拽上传区**: 支持单PDF文件上传
  - **密码设置**: 输入打开密码 + 确认密码，两次输入一致性校验，可显示/隐藏密码
  - **加密强度提示**: 提示密码强度（弱/中/强）
  - **加密操作**: 点击加密按钮显示进度条，使用 jsPDF 的加密功能设置文档打开密码
  - **试用版限制**: 试用版导出的加密PDF每页带水印
  - **下载结果**: 加密完成后显示下载按钮，文件名自动添加"_加密"后缀

### PDF解密页面

- **页面目标**: 用户上传加密PDF，输入正确密码后解密下载
- **功能点**:
  - **拖拽上传区**: 支持加密PDF文件上传，自动检测是否加密
  - **密码输入**: 输入打开密码，可显示/隐藏密码
  - **解密验证**: 使用 pdf.js 尝试用密码打开PDF，验证密码是否正确，错误时提示
  - **解密操作**: 密码正确后点击解密按钮显示进度条，重新生成无密码保护的PDF
  - **试用版限制**: 试用版解密后的PDF每页带水印
  - **下载结果**: 解密完成后显示下载按钮，文件名自动添加"_解密"后缀

## 技术选型说明

- **PDF处理库**: pdf.js（读取/渲染/解密PDF）+ jsPDF（生成/合并/加密PDF）+ pdf-lib（可选，用于更复杂的页面操作），均通过CDN引入
- **UI框架**: React + Tailwind CSS，实现玻璃拟态风格
- **文件处理**: 浏览器原生 File API + Blob + URL.createObjectURL
- **状态管理**: React useState/useContext（全局激活状态、主题状态）
- **打包兼容**: 纯前端单文件/多文件结构，无后端依赖，便于后续用 Electron / Capacitor 打包为桌面/移动应用
- **内置激活码**: 代码注释中预留至少10个有效激活码（格式 PDF-XXXXXXXX，含校验位），供测试和售卖使用

# UI 设计指南

## 1. 设计推导依据

- **参考意图**: Free Direction —— 无参考材料，从产品语义与办公场景出发自主建立视觉系统
- **核心情绪 / 应用类型**: 专业可信、精致克制的办公效率工具，强调隐私安全与操作流畅
- **独特记忆点**: 玻璃拟态卡片叠加微妙渐变光晕，文件拖拽区域带动态边界光效，处理进度以液态填充动画呈现

## 2. Art Direction

- **方向名**: Frosted Glass Office
- **Design Style**: Glassmorphism + Swiss Minimalist —— 毛玻璃营造高级感，瑞士极简保证办公场景的信息清晰度与操作效率
- **DNA 参数**: 圆角 soft (rounded-xl) / 阴影 layered (多层模糊叠加) / 间距 standard (gap-4 / p-6) / 字体方向 几何无衬线 / 装饰手法 渐变光晕 + 细边描边
- **应用类型**: Tool —— 左侧导航 + 右侧工作区的经典工具布局

## 3. Color System

**色彩关系**: 靛蓝主色 + 同色系浅紫反馈底 + 冷白/深空双主题背景，玻璃层通过 backdrop-blur 与半透明白/黑叠加呈现
**配色设计理由**: 靛蓝传达专业、信任与安全，契合办公文档与隐私保护场景；玻璃拟态通过半透明卡片 + 背景渐变光晕实现层次；双主题下保持主色一致，仅调整中性色明度
**主色推导**: 从 PDF 文档的专业感与"隐私安全"语义出发，选定靛蓝色系；通过调整明度衍生 accent 浅底与 border 边界，保持色温统一
**使用比例**: 60% 中性（背景+文字）/ 30% 辅助（玻璃卡片+accent）/ 10% primary（主按钮+激活态+进度条）

| 角色 | CSS 变量 | Tailwind Class | HSL 值 (浅色) | HSL 值 (深色) | 设计说明 |
|---|---|---|---|---|---|
| bg | `--background` | `bg-background` | hsl(220 40% 96%) | hsl(222 47% 11%) | 页面背景，带微妙渐变光晕 |
| card | `--card` | `bg-card` | hsl(0 0% 100% / 0.7) | hsl(222 47% 20% / 0.6) | 玻璃卡片，backdrop-blur-xl |
| text | `--foreground` | `text-foreground` | hsl(222 47% 11%) | hsl(210 40% 98%) | 标题和正文 |
| textMuted | `--muted-foreground` | `text-muted-foreground` | hsl(215 16% 47%) | hsl(215 20% 65%) | 占位符、说明、辅助元信息 |
| primary | `--primary` | `bg-primary` / `text-primary` | hsl(243 75% 59%) | hsl(243 75% 66%) | 主交互、CTA、激活态、进度条 |
| primaryForeground | `--primary-foreground` | `text-primary-foreground` | hsl(0 0% 100%) | hsl(222 47% 11%) | primary 上的文字和图标 |
| accent | `--accent` | `bg-accent` | hsl(243 75% 96%) | hsl(222 47% 25% / 0.5) | hover/focus 浅底、菜单项状态 |
| accentForeground | `--accent-foreground` | `text-accent-foreground` | hsl(243 75% 35%) | hsl(210 40% 98%) | accent 上的文字和图标 |
| border | `--border` | `border-border` | hsl(243 30% 85% / 0.6) | hsl(222 47% 35% / 0.5) | 玻璃卡片边界，半透明 |

**语义色提示**: 成功 hsl(142 71% 45%) —— bg: hsl(142 71% 95%) / border: hsl(142 71% 80%) / text: hsl(142 71% 35%)；错误 hsl(0 84% 60%) —— bg: hsl(0 84% 96%) / border: hsl(0 84% 85%) / text: hsl(0 84% 50%)；警告 hsl(38 92% 50%) —— bg: hsl(38 92% 95%) / border: hsl(38 92% 80%) / text: hsl(38 92% 40%)。语义色饱和度与 primary 对齐 ±10%，避免刺眼。

## 4. 字体与节奏

- **font-display**: Inter —— 几何无衬线，现代清晰，契合办公工具的专业感
- **font-body**: Noto Sans SC + Inter —— 中英文混排舒适，长文本与控件标签均易读
- **字号**: H1 text-2xl ~ text-3xl；H2 text-xl；body text-base；muted text-sm
- **圆角**: 大 (rounded-xl) —— 玻璃卡片柔和圆润，与毛玻璃质感呼应；按钮 rounded-lg；图标按钮 rounded-full

## 5. 全局布局契约

- **Reference Layout Use**: 按需求结构推导 —— 左侧侧边栏导航 + 右侧主工作区 + 顶部状态栏
- **Page / Section Order**: 顶部栏（激活状态+主题切换）→ 左侧导航（5个功能）→ 主内容区（上传区 → 文件列表 → 操作按钮 → 下载区）
- **Standard Content Zone**: Tool max-w-4xl + mx-auto，主工作区卡片居中
- **Shell / Frame Alignment**: 独立滚动 —— 侧边栏固定，主内容区独立滚动
- **Padding & Rhythm**: `px-4 md:px-6 py-6 md:py-8`，保持 8px 倍数节奏
- **Full-bleed Zones**: 页面背景渐变光晕全宽，内容卡片受 Standard Content Zone 约束
- **Local Narrowing**: 激活弹窗、设置表单收窄至 max-w-md
- **Overflow Strategy**: 文件列表超出时使用 `overflow-y-auto`，横向操作栏移动端 `overflow-x-auto`
- **Flexibility Boundary**: 允许移动端侧边栏收起为底部 tab 栏；不允许改变主色、圆角系统和玻璃质感

## 6. 视觉与动效

- **装饰**: 渐变光晕（页面角落 2-3 处柔和彩色光斑）+ 细边描边（玻璃卡片 1px 内发光）
- **阴影/边界**: 中 —— 玻璃卡片带 soft shadow + 内发光边框；悬浮时阴影加深
- **动效**: 精致 —— hover 缩放 1.02 + 阴影加深；文件上传拖拽时边界脉动发光；进度条液态填充动画；页面切换淡入淡出 300ms ease

## 7. 组件原则

- 按钮、输入框、菜单项必须有 Default / Hover / Active / Focus-visible / Disabled 状态
- Primary 按钮用于"开始处理""下载"等主行动；Outline 按钮用于次级操作；Ghost 按钮用于导航项
- 拖拽上传区：默认态（虚线边框+淡底）→ dragover 态（实边+发光+背景加深）→ 已上传态（文件列表取代占位）
- 进度条：渐变填充 + 微光流动效果，完成态变绿色并显示勾选图标
- 激活弹窗：玻璃拟态 + 模糊背景遮罩，输入框带 focus 发光

## 8. Image Direction

- **Image Role**: 背景氛围图
- **Image Art Direction**: 极简抽象渐变光晕，2-3 团柔和彩色光斑（靛蓝+淡紫+浅青）散布在页面角落，透过玻璃卡片产生折射感；低对比度、高模糊度，作为氛围层不抢内容焦点
- **Image Prompt Keywords**: 抽象渐变光晕、靛蓝淡紫色调、柔和模糊光斑、玻璃折射质感、极简背景、低对比度、冷色调、办公科技氛围
- **Image Avoidance**: 避免具象商务人物、文档图标堆砌、高饱和霓虹效果、3D 渲染塑料感、无意义几何形状拼接

## 9. Anti-patterns

- **Split personality**: 深浅主题切换时改变主色或圆角；双主题仅调整中性色明度，primary 与圆角保持一致
- **Glass overload**: 每个元素都加毛玻璃导致层次混乱；只在卡片、弹窗、侧边栏使用玻璃效果，按钮和输入框保持实底
- **Default SaaS drift**: 回到默认蓝色按钮和纯白卡片；用靛蓝主色 + 玻璃质感 + 光晕背景塑造产品识别
- **Invisible interaction**: 只做 hover 不做 focus-visible；所有可交互元素都要有键盘可见的发光描边
- **Mono-hue tyranny**: 主色铺满按钮、tab、图标、边框、链接；按 60-30-10 比例，primary 只用于 CTA、进度条和当前导航高亮
- **Watermark obtrusion**: 试用水印过大或过深影响预览；水印采用低透明度（15%）斜纹平铺，仅在导出文件中出现
